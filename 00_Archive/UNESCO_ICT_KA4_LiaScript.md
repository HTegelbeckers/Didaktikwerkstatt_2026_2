# UNESCO ICT Competency Framework for Teachers

**Knowledge Acquisition – Aspect 4: Application of Digital Skills**

> Teachers use computers, mobile devices, accessible software, and networks for both teaching and learning and management purposes within a framework of **'safe use'**.

---

## Learning Objectives Overview

This module covers competencies **KA.4.a** through **KA.4.n** under the *Knowledge Acquisition* level of the UNESCO ICT-CFT framework.

By the end of this module, teachers should be able to:

- [[x]] Use and explain hardware components and productivity software
- [[x]] Create basic documents, presentations, and graphics
- [[x]] Navigate the Internet safely and effectively
- [[x]] Manage student records using digital tools
- [[x]] Communicate and collaborate using digital technologies

## KA.4.e – Internet Navigation

**Objective:** Navigate the Internet.

> Discuss the purpose and structure of the Internet and the World Wide Web. Use a browser to access popular websites using URLs to navigate the web.

### Key Concepts

| Term | Definition |
|------|-----------|
| URL | Uniform Resource Locator – the address of a web page |
| Browser | Software used to access the web (e.g. Chrome, Firefox) |
| WWW | World Wide Web – a system of interlinked web pages |
| HTTP/HTTPS | Protocols for transferring web data (S = secure) |

### Activity

1. Open a browser and navigate to three different educational websites using URLs.
2. Identify the difference between HTTP and HTTPS addresses.
3. Discuss: why is it important to check whether a website uses HTTPS?

## KA.4.f – Cyber Safety & Media Literacy

**Objective:** Understand the basic principles of cyber safety/security and media and information literacy.

> Demonstrate knowledge of good practices of cyber security and media and information literacy. Ensure safe use of social media and mobile devices.

### Core Principles

1. **Password security** – Use strong, unique passwords for each account.
2. **Phishing awareness** – Recognise suspicious emails and links.
3. **Digital footprint** – Understand that online activity leaves permanent traces.
4. **Media literacy** – Critically evaluate the reliability of online information.
5. **Safe social media use** – Protect privacy settings and personal information.

### Activity

> Scenario: You receive an email claiming to be from your school's IT department asking for your password.

- What are the red flags?
- What should you do?
- How would you teach this concept to students?

## KA.4.g – Search Engines

**Objective:** Use a search engine to find curriculum resources.

> Demonstrate the use of a search engine using simple keyword searches to find subject-related resources. Discuss and consider which keyword strategies elicit the best search results.

### Search Strategies

| Strategy | Example |
|----------|---------|
| Simple keyword | `photosynthesis lesson plan` |
| Phrase search | `"water cycle" primary school` |
| Exclude words | `solar system -astrology` |
| Site-specific | `site:edu climate change` |
| File type | `filetype:pdf maths worksheet` |

### Activity

1. Search for a curriculum resource using a basic keyword.
2. Refine your search using at least two of the strategies above.
3. Compare the quality of results and explain which strategy worked best.

## KA.4.i – Drill-and-Practice Software

**Objective:** Identify and use drill-and-practice software to support learning.

> Analyse the effectiveness of tutorial and drill-and-practice packages, in supporting the acquisition of specific subject matter knowledge. Online software can monitor student usage. Big data can be harnessed to give insights into how students learn.

### Examples of Drill-and-Practice Tools

- **Khan Academy** – adaptive exercises across subjects
- **Duolingo** – language learning with repetitive practice
- **Quizlet** – flashcards and spaced repetition
- **IXL** – curriculum-aligned maths and literacy practice

### Reflection

> "Big data from drill-and-practice tools can reveal patterns in how students learn."

- How can you use usage data from these tools to adapt your teaching?
- What are the privacy implications of collecting student data?

## KA.4.j – Evaluating Educational Software

**Objective:** Identify and evaluate educational software and web resources, and match them to curriculum standards and students' needs.

> Search the Internet to identify appropriate digital resources and tools for specified learning objectives or standards, and analyse these packages for accuracy and curriculum alignment. Discuss assistive technologies enabled by AI for persons with disabilities.

### Evaluation Criteria

Assess any digital resource using these questions:

1. **Accuracy** – Is the content correct and up-to-date?
2. **Curriculum alignment** – Does it match your learning objectives?
3. **Accessibility** – Can all students use it, including those with disabilities?
4. **Usability** – Is the interface intuitive for your students' age group?
5. **Data privacy** – Does it comply with child data protection regulations?

### Activity

Evaluate two educational apps or websites using the criteria above. Present your findings to a peer.

## KA.4.n – Troubleshooting ICT Problems

**Objective:** Troubleshoot ICT problems when technology fails, to ensure minimal disruption to lessons.

> Diagnose common ICT problems – such as lack of power, dropped connectivity, and failed logins – and perform minor maintenance, such as setting up anti-virus utilities.

### Common Problems & Solutions

| Problem | First Steps |
|---------|-------------|
| No power | Check cables, power strip, battery charge |
| No internet | Restart router, check Wi-Fi settings, use mobile hotspot |
| Login failure | Check Caps Lock, reset password, check username |
| Projector not displaying | Check cable connection, switch input source |
| Slow computer | Close unused apps, clear browser cache, restart |
| Virus/malware suspected | Run anti-virus scan, disconnect from network |

### Activity

Work in pairs to **role-play a classroom scenario** where technology fails. One person plays the teacher, the other the IT support. Diagnose and resolve the problem in under 3 minutes.
